A reproduction of **information-driven-bar trading** on cryptocurrency data: instead of sampling the
market on a fixed time grid, sample it on **price movement** — **CUSUM** and **range** bars — then
label each bar with the **triple-barrier** method and train a **ResNet-LSTM** (and TSMixer) classifier
under a quarterly **expanding-window** protocol, and backtest with a 60/40 probability band.

**Headline target (the cell this pipeline is built to reproduce):** ETH, 2% CUSUM bars, triple-barrier
labels ≈ **+91.6% net P/L, 1.42 Sharpe** over the 2022-Q2 … 2023-Q2 out-of-sample window
(buy-&-hold lost ~44% over the same period).

Methods follow López de Prado (CUSUM filter, triple-barrier labeling) with 1-minute input bars
(Borges & Neves) and a ResNet-LSTM sequence model.

## Quickstart

Requires Python 3.12 with `torch` (CUDA for training), `pandas`, `polars`, `numpy`, `scikit-learn`, `pyarrow`.

```bash
# 1) build the data (downloads ~1.4 GB of klines, then bars, then feature datasets)
python3 data_pipeline/ingest.py
python3 data_pipeline/build_bars.py
python3 model/build_dataset.py
# 2) train the v1 grid (GPU) and evaluate
python3 model/train.py --grid
python3 model/evaluate.py
```

`train.py` auto-uses CUDA when available (the local pipeline was developed CPU-only; full training is
meant for a GPU box). Predictions and metrics land in `results/model/`.

## Method notes

- **Bars** sample on accumulated (CUSUM) or point-to-point (range) price moves at 1/2/3% thresholds.
- **Triple-barrier** labels: up/down by which of ±barrier is touched first within a 24-bar timeout
  (segment-aware — the forward look never crosses a data gap).
- **Expanding window**: for each test quarter, validation = prior quarter, train = everything before;
  sampling/labeling params are fixed across quarters (no per-quarter re-tuning).
- **Leakage discipline**: the scaler is fit on train only; a 96-bar window never crosses a data gap or
  the train/test boundary.
